# Package Indexes

Tests for package index configuration in dependency resolution.

## No index

### No index in requirements file

<!-- from pip_compile.rs::no_index_requirements_txt -->

Using `--no-index` prevents package lookup from any index.

```toml
# file: requirements.in
--no-index
tqdm
```

```console
$ uv pip compile requirements.in
success: false
exit_code: 1
----- stdout -----

----- stderr -----
  × No solution found when resolving dependencies:
  ╰─▶ Because tqdm was not found in the provided package locations and you require tqdm, we can conclude that your requirements are unsatisfiable.

      hint: Packages were unavailable because index lookups were disabled and no additional package locations were provided (try: `--find-links <uri>`)
```

### Install with no index

<!-- from pip_install.rs::install_no_index -->

Install fails when `--no-index` is specified and no local sources available.

```console
$ uv pip install Flask --no-index
success: false
exit_code: 1
----- stdout -----

----- stderr -----
  × No solution found when resolving dependencies:
  ╰─▶ Because flask was not found in the provided package locations and you require flask, we can conclude that your requirements are unsatisfiable.

      hint: Packages were unavailable because index lookups were disabled and no additional package locations were provided (try: `--find-links <uri>`)
```

### No index with version specifier

<!-- from pip_install.rs::install_no_index_version -->

Error includes version specifier when specified.

```console
$ uv pip install Flask==3.0.0 --no-index
success: false
exit_code: 1
----- stdout -----

----- stderr -----
```

## Index URL configuration

### Index URL in requirements file

<!-- from pip_compile.rs::index_url_in_requirements -->

An index URL specified in requirements file is used for resolution.

```toml
# file: requirements.in
--index-url https://astral-sh.github.io/pytorch-mirror/whl
anyio<4
```

```console
$ uv pip compile requirements.in
success: false
exit_code: 1
----- stdout -----

----- stderr -----
  × No solution found when resolving dependencies:
  ╰─▶ Because anyio was not found in the package registry and you require anyio<4, we can conclude that your requirements are unsatisfiable.
```

### Command line overrides requirements file

<!-- from pip_compile.rs::index_url_from_command_line -->

Command line `--index-url` takes precedence over requirements file.

```toml
# file: requirements.in
--index-url https://astral-sh.github.io/pytorch-mirror/whl
anyio<4
```

```console
$ uv pip compile requirements.in --index-url https://pypi.org/simple
success: true
exit_code: 0
----- stdout -----
# This file was autogenerated by uv via the following command:
#    uv --cache-dir [CACHE_DIR] pip compile requirements.in
anyio==3.7.1
    # via -r requirements.in
idna==3.6
    # via anyio
sniffio==1.3.1
    # via anyio

----- stderr -----
Resolved 3 packages in [TIME]
```

### Command line overrides requirements file index

<!-- from pip_compile.rs::index_url_requirements_txt -->

The `--index-url` from command line overrides the one in requirements file.

```toml
# file: requirements.in
--index-url https://google.com
tqdm
```

```console
$ uv pip compile requirements.in --index-url https://pypi.org/simple
success: true
exit_code: 0
----- stdout -----
# This file was autogenerated by uv via the following command:
#    uv --cache-dir [CACHE_DIR] pip compile requirements.in
tqdm==4.66.2
    # via -r requirements.in

----- stderr -----
Resolved 1 package in [TIME]
```

### Conflicting index URLs

<!-- from pip_compile.rs::conflicting_index_urls_requirements_txt -->

Different `--index-url` in multiple requirements files raises an error.

```toml
# file: requirements.in
--index-url https://google.com
tqdm
```

```toml constraints.in
--index-url https://wikipedia.org
flask
```

```console
$ uv pip compile requirements.in --constraint constraints.in
success: false
exit_code: 2
----- stdout -----

----- stderr -----
error: Multiple index URLs specified: `https://google.com/` vs. `https://wikipedia.org/`
```

### Matching index URLs

<!-- from pip_compile.rs::matching_index_urls_requirements_txt -->

Same `--index-url` in multiple files is allowed.

```toml
# file: requirements.in
--index-url https://pypi.org/simple
```

```toml constraints.in
--index-url https://pypi.org/simple
```

```console
$ uv pip compile requirements.in --constraint constraints.in
success: true
exit_code: 0
----- stdout -----
# This file was autogenerated by uv via the following command:
#    uv --cache-dir [CACHE_DIR] pip compile requirements.in --constraint constraints.in

----- stderr -----
Resolved in [TIME]
```

## Extra index URL

### Extra index URL priority

<!-- from pip_install.rs::install_extra_index_url_has_priority -->

The `--extra-index-url` is checked before `--index-url`.

```console
$ uv pip install --index-url https://test.pypi.org/simple --extra-index-url https://pypi.org/simple black==24.2.0 --no-deps --exclude-newer 2024-03-09T00:00:00Z
success: true
exit_code: 0
----- stdout -----

----- stderr -----
Resolved 1 package in [TIME]
Prepared 1 package in [TIME]
Installed 1 package in [TIME]
 + black==24.2.0
```

## Index strategies

### First match (default)

<!-- from pip_compile.rs::compile_index_url_first_match_base -->

By default, uv uses `first-match` strategy which fails if the first index with the package doesn't
have the requested version.

```toml
# file: requirements.in
jinja2==3.1.0
```

```console
$ uv pip compile --index-url https://pypi.org/simple --extra-index-url https://astral-sh.github.io/pytorch-mirror/whl/cpu requirements.in --no-deps --exclude-newer 2025-01-30T00:00:00Z
success: false
exit_code: 1
----- stdout -----

----- stderr -----
  × No solution found when resolving dependencies:
  ╰─▶ Because there is no version of jinja2==3.1.0 and you require jinja2==3.1.0, we can conclude that your requirements are unsatisfiable.

      hint: `jinja2` was found on https://astral-sh.github.io/pytorch-mirror/whl/cpu, but not at the requested version (jinja2==3.1.0). A compatible version may be available on a subsequent index (e.g., https://pypi.org/simple). By default, uv will only consider versions that are published on the first index that contains a given package, to avoid dependency confusion attacks. If all indexes are equally trusted, use `--index-strategy unsafe-best-match` to consider all versions from all indexes, regardless of the order in which they were defined.
```

### First match with marker

<!-- from pip_compile.rs::compile_index_url_first_match_marker -->

First-match strategy applies even with platform markers in universal resolution.

```toml
# file: requirements.in
jinja2==3.1.0 ; sys_platform == 'linux'
```

```console
$ uv pip compile --universal --index-url https://pypi.org/simple --extra-index-url https://astral-sh.github.io/pytorch-mirror/whl/cpu requirements.in --no-deps --exclude-newer 2025-01-30T00:00:00Z
success: false
exit_code: 1
----- stdout -----

----- stderr -----
  × No solution found when resolving dependencies:
  ╰─▶ Because there is no version of jinja2{sys_platform == 'linux'}==3.1.0 and you require jinja2{sys_platform == 'linux'}==3.1.0, we can conclude that your requirements are unsatisfiable.

      hint: `jinja2` was found on https://astral-sh.github.io/pytorch-mirror/whl/cpu, but not at the requested version (jinja2==3.1.0). A compatible version may be available on a subsequent index (e.g., https://pypi.org/simple). By default, uv will only consider versions that are published on the first index that contains a given package, to avoid dependency confusion attacks. If all indexes are equally trusted, use `--index-strategy unsafe-best-match` to consider all versions from all indexes, regardless of the order in which they were defined.
```

### First match with no versions

<!-- from pip_compile.rs::compile_index_url_first_match_all_versions -->

If a package exists on an index but with no compatible versions, resolution fails even if other
indexes have the package.

```toml
# file: requirements.in
pandas
```

```console
$ uv pip compile --index-url https://pypi.org/simple --extra-index-url https://test.pypi.org/simple requirements.in --no-deps
success: false
exit_code: 1
----- stdout -----

----- stderr -----
  × No solution found when resolving dependencies:
  ╰─▶ Because there are no versions of pandas and you require pandas, we can conclude that your requirements are unsatisfiable.
```

### Unsafe any match fallback

<!-- from pip_compile.rs::compile_index_url_fallback -->

With `--index-strategy unsafe-any-match`, resolution falls back to other indexes.

```toml
# file: requirements.in
jinja2==3.1.0
```

```console
$ uv pip compile --index-strategy unsafe-any-match --index-url https://pypi.org/simple --extra-index-url https://astral-sh.github.io/pytorch-mirror/whl/cpu requirements.in --no-deps --exclude-newer 2025-01-30T00:00:00Z
success: true
exit_code: 0
----- stdout -----
# This file was autogenerated by uv via the following command:
#    uv --cache-dir [CACHE_DIR] pip compile --index-strategy unsafe-any-match requirements.in --no-deps
jinja2==3.1.0
    # via -r requirements.in

----- stderr -----
Resolved 1 package in [TIME]
```

### Unsafe any match prefers primary

<!-- from pip_compile.rs::compile_index_url_fallback_prefer_primary -->

With `unsafe-any-match`, the preferred index version is used even if newer versions exist elsewhere.

```toml
# file: requirements.in
anyio
```

```console
$ uv pip compile --index-strategy unsafe-any-match --index-url https://pypi.org/simple --extra-index-url https://test.pypi.org/simple requirements.in --no-deps
success: true
exit_code: 0
----- stdout -----
# This file was autogenerated by uv via the following command:
#    uv --cache-dir [CACHE_DIR] pip compile --index-strategy unsafe-any-match requirements.in --no-deps
anyio==3.5.0
    # via -r requirements.in

----- stderr -----
Resolved 1 package in [TIME]
```

### Unsafe best match highest

<!-- from pip_compile.rs::compile_index_url_unsafe_highest -->

With `--index-strategy unsafe-best-match`, the highest compatible version is used regardless of
which index it comes from.

```toml
# file: requirements.in
anyio
```

```console
$ uv pip compile --index-strategy unsafe-best-match --index-url https://pypi.org/simple --extra-index-url https://test.pypi.org/simple requirements.in --no-deps
success: true
exit_code: 0
----- stdout -----
# This file was autogenerated by uv via the following command:
#    uv --cache-dir [CACHE_DIR] pip compile --index-strategy unsafe-best-match requirements.in --no-deps
anyio==4.3.0
    # via -r requirements.in

----- stderr -----
Resolved 1 package in [TIME]
```

### Unsafe best match lowest

<!-- from pip_compile.rs::compile_index_url_unsafe_lowest -->

With `unsafe-best-match` and `--resolution lowest`, the lowest compatible version is used.

```toml
# file: requirements.in
anyio<100
```

```console
$ uv pip compile --resolution lowest --index-strategy unsafe-best-match --index-url https://pypi.org/simple --extra-index-url https://test.pypi.org/simple requirements.in --no-deps
success: true
exit_code: 0
----- stdout -----
# This file was autogenerated by uv via the following command:
#    uv --cache-dir [CACHE_DIR] pip compile --resolution lowest --index-strategy unsafe-best-match requirements.in --no-deps
anyio==1.0.0
    # via -r requirements.in

----- stderr -----
warning: The direct dependency `anyio` is unpinned. Consider setting a lower bound when using `--resolution lowest` or `--resolution lowest-direct` to avoid using outdated versions.
Resolved 1 package in [TIME]
```

## Emit index URL

### Emit default index URL

<!-- from pip_compile.rs::emit_index_urls -->

The `--emit-index-url` flag includes the index URL in output.

```toml
# file: requirements.in
iniconfig
```

```console
$ uv pip compile requirements.in --emit-index-url --default-index https://test.pypi.org/simple/
success: true
exit_code: 0
----- stdout -----
# This file was autogenerated by uv via the following command:
#    uv --cache-dir [CACHE_DIR] pip compile requirements.in --emit-index-url --default-index https://test.pypi.org/simple/
--index-url https://test.pypi.org/simple/

iniconfig==2.0.0
    # via -r requirements.in

----- stderr -----
Resolved 1 package in [TIME]
```

### Emit extra index URL

<!-- from pip_compile.rs::emit_index_urls -->

When `--index` is used with `--emit-index-url`, both primary and extra indexes are emitted.

```toml
# file: requirements.in
iniconfig
```

```console
$ uv pip compile requirements.in --emit-index-url --index https://test.pypi.org/simple/
success: true
exit_code: 0
----- stdout -----
# This file was autogenerated by uv via the following command:
#    uv --cache-dir [CACHE_DIR] pip compile requirements.in --emit-index-url --index https://test.pypi.org/simple/
--index-url https://pypi.org/simple
--extra-index-url https://test.pypi.org/simple/

iniconfig==2.0.0
    # via -r requirements.in

----- stderr -----
Resolved 1 package in [TIME]
```


Without trailing slash:

