# pip compile Errors

Tests for error handling in `uv pip compile`.

```toml
# mdtest

[environment]
python-version = "3.12"
required-features = "pypi"
```

## Missing input

### Missing requirements file

<!-- from pip_compile.rs::missing_requirements_in -->

Running with a missing requirements file shows an error.

```console
$ uv pip compile requirements.in
success: false
exit_code: 2
----- stdout -----

----- stderr -----
error: File not found: `requirements.in`
```

## Unsolvable requirements

### Conflicting direct dependencies

<!-- from pip_compile.rs::compile_unsolvable_requirements -->

Compile requirements that cannot be solved due to conflicting direct dependencies.

```toml
# file: pyproject.toml
[build-system]
requires = ["setuptools>=42"]

[project]
name = "my-project"
version = "0.1.0"
dependencies = ["anyio==3.7.0", "anyio==4.0.0"]
```

```console
$ uv pip compile pyproject.toml
success: false
exit_code: 1
----- stdout -----

----- stderr -----
  × No solution found when resolving dependencies:
  ╰─▶ Because my-project depends on anyio==3.7.0 and anyio==4.0.0, we can conclude that your requirements are unsatisfiable.
```

### Version not available

<!-- from pip_compile.rs::compile_unsolvable_requirements_version_not_available -->

Compile requirements with a version that does not exist.

```toml
# file: pyproject.toml
[build-system]
requires = ["setuptools>=42"]

[project]
name = "my-project"
version = "0.1.0"
dependencies = ["anyio==300.1.4"]
```

```console
$ uv pip compile pyproject.toml
success: false
exit_code: 1
----- stdout -----

----- stderr -----
  × No solution found when resolving dependencies:
  ╰─▶ Because there is no version of anyio==300.1.4 and my-project depends on anyio==300.1.4, we can conclude that your requirements are unsatisfiable.
```

## Yanked versions

### Direct yanked version

<!-- from pip_compile.rs::compile_yanked_version_direct -->

Resolving a yanked version directly emits a warning.

```toml
# file: requirements.in
attrs==21.1.0
```

```console
$ uv pip compile requirements.in
success: true
exit_code: 0
----- stdout -----
# This file was autogenerated by uv via the following command:
#    uv --cache-dir [CACHE_DIR] pip compile requirements.in
attrs==21.1.0
    # via -r requirements.in

----- stderr -----
Resolved 1 package in [TIME]
warning: `attrs==21.1.0` is yanked (reason: "Installable but not importable on Python 3.4.")
```

### Indirect yanked version

<!-- from pip_compile.rs::compile_yanked_version_indirect -->

Resolving a yanked version indirectly fails resolution.

```toml
# file: requirements.in
attrs>20.3.0,<21.2.0
```

```console
$ uv pip compile requirements.in
success: false
exit_code: 1
----- stdout -----

----- stderr -----
  × No solution found when resolving dependencies:
  ╰─▶ Because only the following versions of attrs are available:
          attrs<=20.3.0
          attrs==21.1.0
          attrs>=21.2.0
      and attrs==21.1.0 was yanked (reason: Installable but not importable on Python 3.4), we can conclude that attrs>20.3.0,<21.2.0 cannot be used.
      And because you require attrs>20.3.0,<21.2.0, we can conclude that your requirements are unsatisfiable.
```

## Invalid Python version

### Invalid Python version format

<!-- from pip_compile.rs::compile_python_invalid_version -->

Invalid Python version format produces an error.

```toml
# file: requirements.in
black==23.10.1
```

```console
$ uv pip compile requirements.in --python-version 3.7.x
success: false
exit_code: 2
----- stdout -----

----- stderr -----
error: invalid value '3.7.x' for '--python-version <PYTHON_VERSION>': Python version `3.7.x` could not be parsed: after parsing `3.7`, found `.x`, which is not part of a valid version

For more information, try '--help'.
```

### Development version not allowed

<!-- from pip_compile.rs::compile_python_dev_version -->

Development versions are not allowed for `--python-version`.

```console
$ uv pip compile requirements.in --python-version 3.7-dev
success: false
exit_code: 2
----- stdout -----

----- stderr -----
error: invalid value '3.7-dev' for '--python-version <PYTHON_VERSION>': Python version `3.7-dev` is a development release

For more information, try '--help'.
```

### Conflicting Python argument and flag

<!-- from pip_compile.rs::compile_python_conflicts -->

Using both `--python` and `-p` fails because they are the same argument.

```toml
# file: requirements.in
black==23.10.1
```

```console
$ uv pip compile requirements.in --python 3.12 -p 3.12
success: false
exit_code: 2
----- stdout -----

----- stderr -----
error: the argument '--python <PYTHON>' cannot be used multiple times

Usage: uv pip compile [OPTIONS] <SRC_FILE|--group <GROUP>>

For more information, try '--help'.
```

## URL errors

### Invalid URL file extension

<!-- from pip_compile.rs::invalid_tool_uv_sources -->

A direct URL with an unsupported file extension fails.

```toml
# file: pyproject.toml
[project]
name = "project"
version = "0.0.0"
dependencies = [
  "urllib3 @ https://files.pythonhosted.org/packages/a2/73/a68704750a7679d0b6d3ad7aa8d4da8e14e151ae82e6fee774e6e0d05ec8/urllib3-2.2.1-py3-none-any.tar.baz",
]
```

```console
$ uv pip compile pyproject.toml
success: false
exit_code: 2
----- stdout -----

----- stderr -----
error: Failed to parse metadata from built wheel
  Caused by: Expected direct URL (`https://files.pythonhosted.org/packages/a2/73/a68704750a7679d0b6d3ad7aa8d4da8e14e151ae82e6fee774e6e0d05ec8/urllib3-2.2.1-py3-none-any.tar.baz`) to end in a supported file extension: `.whl`, `.tar.gz`, `.zip`, `.tar.bz2`, `.tar.lz`, `.tar.lzma`, `.tar.xz`, `.tar.zst`, `.tar`, `.tbz`, `.tgz`, `.tlz`, or `.txz`

```

### Unsupported URL scheme

<!-- from pip_compile.rs::unsupported_scheme -->

Using an unsupported URL scheme like `bzr` fails with a helpful error.

```toml
# file: requirements.in

```

```console
$ uv pip compile requirements.in
success: false
exit_code: 2
----- stdout -----

----- stderr -----
error: Couldn't parse requirement in `requirements.in` at position 0
  Caused by: Unsupported URL prefix `bzr` in URL: `bzr+https://example.com/anyio` (Bazaar is not supported)
anyio @ bzr+https://example.com/anyio
        ^^^^^^^^^^^^^^^^^^^^^^^^^^^^^
```

### Missing environment variable syntax

<!-- from pip_compile.rs::error_missing_unnamed_env_var -->

A requirements file with unsubstituted environment variable syntax fails with a parse error.


```console
$ uv pip compile requirements.in

----- stderr -----
error: Couldn't parse requirement in `requirements.in` at position 0
  Caused by: Expected package name starting with an alphanumeric character, found `$`
${URL}
^
```

### Python version too old for package

<!-- from pip_compile.rs::compile_python_37 -->

Resolving a package that requires a newer Python version fails with a hint.

```toml
# file: requirements.in
black==23.10.1
```

```console
$ uv pip compile requirements.in --python-version 3.7
success: false
exit_code: 1
----- stdout -----

----- stderr -----
warning: The requested Python version 3.7 is not available; 3.12.[X] will be used to build dependencies instead.
  × No solution found when resolving dependencies:
  ╰─▶ Because the requested Python version (>=3.7) does not satisfy Python>=3.8 and black==23.10.1 depends on Python>=3.8, we can conclude that black==23.10.1 cannot be used.
      And because you require black==23.10.1, we can conclude that your requirements are unsatisfiable.

      hint: The `--python-version` value (>=3.7) includes Python versions that are not supported by your dependencies (e.g., black==23.10.1 only supports >=3.8). Consider using a higher `--python-version` value.
```

### Current Python version incompatible with package

<!-- from pip_compile.rs::compile_enumerate_no_versions -->

Resolving a package that requires a newer Python version than the interpreter.

```toml
# mdtest

[environment]
exclude-newer = "2024-12-01"
python-version = "3.10"
```

```toml
# file: requirements.in
rooster-blue
```

```console
$ uv pip compile requirements.in
success: false
exit_code: 1
----- stdout -----

----- stderr -----
  × No solution found when resolving dependencies:
  ╰─▶ Because the current Python version (3.10.[X]) does not satisfy Python>=3.11,<4.0 and all versions of rooster-blue depend on Python>=3.11,<4.0, we can conclude that all versions of rooster-blue cannot be used.
      And because you require rooster-blue, we can conclude that your requirements are unsatisfiable.
```

### Wheel name mismatch

<!-- from pip_compile.rs::requirement_wheel_name_mismatch -->

Requesting a package with a name that doesn't match the wheel filename fails.

```toml
# file: requirements.in
dateutil @ https://files.pythonhosted.org/packages/ec/57/56b9bcc3c9c6a792fcbaf139543cee77261f3651ca9da0c93f5c1221264b/python_dateutil-2.9.0.post0-py2.py3-none-any.whl
```

```console
$ uv pip compile requirements.in
success: false
exit_code: 2
----- stdout -----

----- stderr -----
error: Requested package name `dateutil` does not match `python-dateutil` in the distribution filename: https://files.pythonhosted.org/packages/ec/57/56b9bcc3c9c6a792fcbaf139543cee77261f3651ca9da0c93f5c1221264b/python_dateutil-2.9.0.post0-py2.py3-none-any.whl
```

## Output file errors

### Invalid output file format

<!-- from pip_compile.rs::compile_invalid_output_file -->

Using `pyproject.toml` as output file fails since only requirements.txt format is supported.

```toml
# file: requirements.in
anyio==3.7.0
```

```console
$ uv pip compile requirements.in -o pyproject.toml
success: false
exit_code: 2
----- stdout -----

----- stderr -----
error: `pyproject.toml` is not a supported output format for `uv pip compile` (only `requirements.txt`-style output is supported)
```

### Invalid PEP 751 filename

<!-- from pip_compile.rs::pep_751_filename -->

PEP 751 output must use the `pylock.*.toml` naming convention.

```toml
# file: requirements.txt
iniconfig
```

```console
$ uv pip compile requirements.txt --universal --format pylock.toml -o test.toml
success: false
exit_code: 2
----- stdout -----

----- stderr -----
error: Expected the output filename to start with `pylock.` and end with `.toml` (e.g., `pylock.toml`, `pylock.dev.toml`); `test.toml` won't be recognized as a `pylock.toml` file in subsequent commands
```
